# Mystic Dungeon 設計仕様書

本仕様書は、AIコーディングエージェントが本ゲームを再現するための
詳細設計書である。以下の仕様に従い、Python + curses で実装すること。

---

## 1. ゲーム概要

ターン制ダンジョン探索RPG。プレイヤーは🧙となり、ランダム生成された
ダンジョンを探索し、モンスターと戦いながら伝説の装備を集める。

- ターン制（プレイヤーの行動後にモンスターが行動）
- 視野（FOV）制御により未探索エリアは見えない
- マップは部屋＋通路の自動生成
- 3段階の難易度選択
- セーブ3スロット、StrongNewGame機能
- ターン制テキストバトル（武器選択式）
- ショップシステム（購入/売却）
- レアリティ付き装備品DB
- 「不思議のダンジョン」シリーズをモチーフとした名称・システム

## 4. マップ生成アルゴリズム

### 4.1. 部屋生成

1. MAP_W × MAP_H を壁で初期化
2. 試行回数 = 目標部屋数 × 3（最大）
3. 各試行:
   a. 部屋サイズ: [min_room, max_room] からランダム
   b. 位置: マップ端から1マス内側にランダム
   c. 既存の部屋と交差判定 → 交差しなければ追加
4. 部屋が0個の場合: 6x5の部屋を強制配置
5. 各部屋の内部を床に変換

### 4.2. 通路接続

1. 部屋を生成順に隣接ペア(a, b)とする
2. 各ペアをL字型通路で接続:
   - 50%: 水平→垂直
   - 50%: 垂直→水平
3. 通路の各セルを床に変換（マップ端の壁は維持）

### 4.3. 配置

**プレイヤー**: 最初の部屋の中央
**階段**: 最後の部屋の中央（部屋数が2以上の場合のみ）
**ショップ**: 中間の部屋のいずれかに35%の確率で出現（部屋数が3以上の場合）
**モンスター**: 各部屋（最初を除く）に 0〜spawn_count 体をランダム配置。
  パラメータ: dlvl_scale = 1.0 + (dlvl - 1) * 0.3
  HP/攻撃力/防御力 *= dlvl_scale * dmg_scale
  経験値 *= dlvl_scale
**レアベアラー**: 生成されたモンスターの中からランダムに1体を選択。
  rare_bearer = True に設定。このモンスターを倒すとレアアイテム確定ドロップ。
**アイテム**: 各部屋（最初を除く）に金貨のみ配置。金額 = random(1, 10*dlvl)。

**床に武器・防具・回復アイテムは落ちていない。** 全て敵ドロップかショップで入手。

## 5. 視野 (FOV)

### アルゴリズム

Bresenhamの線分アルゴリズムによるレイキャスティング。

```
function has_los(x0, y0, x1, y1):
    dx = abs(x1 - x0)
    dy = abs(y1 - y0)
    sx = (x0 < x1) ? 1 : -1
    sy = (y0 < y1) ? 1 : -1
    err = dx - dy
    x = x0, y = y0
    loop:
        if (x, y) == (x1, y1): return true
        if (x, y) != (x0, y0) AND map[y][x] == 壁: return false
        if 2*err > -dy: err -= dy; x += sx
        if 2*err < dx: err += dx; y += sy
```

visibleは毎ターンリセット。exploredは累積。

### 6.1. クラス定義

```
Entity:
    x, y: 座標

    color: curses色ペア番号
    blocks: 移動ブロックするか
    name: 表示名

    xp: 経験値

    armor_item: 装備中の防具Item
```

| Legendary | Dragon Mail, Aegis |

### 7.4. インベントリ操作

**最大10個まで**所持可能。

ピックアップ:
- 金: 自動で所持金に加算。Itemは消滅。
- ショップ: `s`キーでショップ画面を開く。
- 階段: フロア移動。

装備:
- インベントリ画面でアイテム番号選択 → 装備（武器か防具のみ）
- 武器と防具は同時装備可能
- 現在の装備品はインベントリに戻り、選択したアイテムが装備される

ドロップ（`d`キー）:
- インベントリ画面で番号選択 → アイテムをその場にドロップ
- 装備中のアイテムをドロップ → 自動的に装備解除

### 7.5. アイテム入手

- **マップ上**: 金貨のみ。自動取得。
- **敵ドロップ**: 倒した敵から40%で武器/防具/回復アイテムがドロップ。
  レアリティ抽選: Common 40%, Uncommon 30%, Rare 20%, Legendary 10%
- **ショップ**: フロアに出現するショップで購入/売却。
- **レアベアラー**: 各フロアに1体いる。倒すと Rare or Legendary 確定ドロップ。

## 8. 戦闘システム

### 8.1. バトル画面

敵のいるマスに移動、または敵がプレイヤーに隣接した時に遷移。
接敵側が常に先攻。

### 8.2. 戦闘の流れ

1. 武器選択（`e`=装備武器, `1-9`=インベントリ, `f`=素手）
2. 攻撃側のダメージ計算
3. 相手が生存していれば反撃
4. どちらかが倒れるまで繰り返し

### 8.3. ダメージ計算式

```
攻撃力計算:
  if 武器.attack_type == 'physical':
    atk_power = 攻撃側.patk + 武器.patk
  else:
    atk_power = 攻撃側.matk + 武器.matk

弱点適用（防御計算の前に乗算）:
  if 防御側.weakness == 攻撃属性:
    atk_power *= 1.5

会心率 = 攻撃側.crit_rate + 武器.crit_rate
会心倍率 = 攻撃側.crit_dmg + 武器.crit_dmg - 1.0

if クリティカル発生:
  ダメージ = int(atk_power * 会心倍率)  // 防御無視
else:
  防御力 = 防御側.mdef (魔法) or 防御側.pdef (物理)
  ダメージ = max(3, atk_power - 防御力)
```

最低ダメージは3（非会心時）。

### 8.4. レベルアップ

xp >= max_hp * 2 で発動:
- max_hp += 5
- patk += 1
- pdef += 1
- mdef += 1

### 8.5. モンスターの行動

- プレイヤーが15マス以内かつ視野内 → 接近（greedy movement）
- 隣接 → バトル画面に遷移（モンスター先攻）

## 9. ショップシステム

### 9.1. 出現

- フロア生成時に35%の確率で1部屋にショップタイル（🏪）が設置
- タイルの上で`s`キーを押すとショップ画面を開く

### 9.2. 購入

- 3〜5個のランダムな武器/防具を販売
- レアリティ抽選: Common 30%, Uncommon 30%, Rare 25%, Legendary 15%
- 購入価格 = 50 + bonus * 30（武器）、40 + bonus * 40（防具）

### 9.3. 売却

- インベントリの武器/防具/回復アイテムを売却可能
- 売却価格 = max(1, アイテム.value // 2)

## 10. レアリティシステム

### 10.1. レアリティ定義

| 値 | 名称 | タグ | ボーナス範囲 | ドロップ重み |
|-----|------|------|-------------|-------------|
| 0 | Common | （なし） | +0 | 40 |
| 1 | Uncommon | `+` | 0〜+1 | 30 |
| 2 | Rare | `★` | +1〜+3+dlvl/2 | 20 |
| 3 | Legendary | `✦` | +2〜+5+dlvl/2 | 10 |

### 10.2. レアベアラー

- 各フロア生成時に1体のモンスターがレアベアラーに選ばれる
- レアベアラーを倒すと Rare or Legendary（60%/40%）確定ドロップ
- レアベアラーが倒されるまで、他の敵のレア/レジェンドドロップ確率も3倍にブースト

### 10.3. フロア保証

各フロアで最低1回は Rare 以上のドロップが発生する。
レアベアラーが倒されなくても、いずれかの敵からレア以上が出る確率が
上がり続ける。

## 12. セーブ／ロードシステム

### 12.1. 保存形式

JSONファイル。ファイル名: `rogue_save_1.json` 〜 `rogue_save_3.json`
versionフィールドで互換性管理（現在: 4）。

保存データ構造:
```json
{
  "version": 4,
  "diff": 2,
  "dlvl": 5,
  "MAP_W": 40,
  "MAP_H": 24,
  "player": {
    "x": 10, "y": 5,
    "hp": 25, "max_hp": 35,
    "xp": 45, "gold": 120,
    "patk": 5, "matk": 3,
    "pdef": 2, "mdef": 3,
    "crit_rate": 0.05, "crit_dmg": 1.5,
    "inventory": [ /* Item */ ],
    "weapon_item": { /* Item or null */ },
    "armor_item": { /* Item or null */ }
  },
  "map": [ [35, 35, 46, ...], ... ],
  "explored": [ [true, false, ...], ... ],
  "visible": [ [false, false, ...], ... ],
  "entities": [
    { "name": "Goblin", "x": 12, "y": 8,
      "hp": 30, "max_hp": 30,
      "patk": 5, "matk": 2, "pdef": 2, "mdef": 1,
      "crit_rate": 0.05, "crit_dmg": 1.5,
      "weakness": "physical", "xp": 12,
      "rare_bearer": false }
  ],
  "items": [ /* Item */ ],
  "messages": [ "...", "..." ],
  "rare_dropped_this_floor": false
}
```

Itemのシリアライズ:
```json
{
  "x": 0, "y": 0, "emoji": "⚔️", "name": "Short Sword+1",
  "type": "weapon", "value": 60,
  "patk": 3, "matk": 0,
  "pdef": 0, "mdef": 0,
  "crit_rate": 0.05, "crit_dmg": 1.5,
  "attack_type": "physical",
  "rarity": 1
}
```

### 12.2. セーブ操作

- **S** (Shift+s): 現在のスロットに上書き保存して続行
- **Q** (Shift+q): 現在のスロットに上書き保存して終了
- **q**: セーブせず終了

### 12.3. ロード操作

起動時、セーブファイルの存在確認:
- あり → スロット選択画面
- なし → 難易度選択画面

### 12.4. セーブスロット情報取得

各スロットのファイル存在確認:
- ファイルなし → None
- パース成功 → { dlvl, hp, max_hp, diff }
- パース失敗 → { corrupted: true }

## 13. Strong New Game

ゲームオーバー画面で選択すると、以下の状態を維持して新規ゲーム開始:

- max_hp, hp（フル回復）
- patk, matk, pdef, mdef
- crit_rate, crit_dmg
- gold, xp
- inventory（アイテムリストのコピー）
- weapon_item, armor_item

これらを `saved` 変数に保持し、次の `New Game` 時に適用。
難易度とマップは新規生成。

## 16. 実装時の注意点

1. 絵文字の表示幅は端末の実装に依存する。addstrで行単位に描画し、
   各行が画面幅を超えないようスライスすること。

2. cursesのinit_pairの引数順は（pair_number, foreground, background）。
   -1を背景色に指定すると端末デフォルト背景になる。

3. セーブデータのバージョンチェック: 将来の互換性のため、
   versionフィールドでデータ構造のバージョンを管理する。

4. ターン消費ルール:
   - 移動が成功した場合のみモンスターターン実行
   - 壁にぶつかったなどはターン消費しない
   - 休息(.)、拾う(Space)、階段(>)は常にターン消費

5. マップデータは `ord('#')` と `ord('.')` の整数値で保持。
   描画時に絵文字に変換。

6. 階段による階層移動時: マップ再生成後、rare_dropped_this_floor は
   False にリセット。

7. 武器の射程概念は存在しない。全ての攻撃は隣接マスの敵に対して行う。
